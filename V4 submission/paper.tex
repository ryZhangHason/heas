\begin{abstract}
HEAS is a reusable framework for \emph{ODE-style and tabular} simulation-based
policy search that standardizes metric extraction across evolutionary search,
tournament evaluation, and statistical validation. The framework addresses a
recurring ABM integration problem in this model family: researchers often rebuild
100--200 lines of coupling code per project, and the same outcome metric can
silently diverge across optimization, comparison, and inference code paths.
HEAS addresses this through a four-component design: (C1)~a
Layer/Stream/Arena composition model; (C2)~a uniform \texttt{metrics\_episode()}
contract consumed identically by the evolutionary algorithm, tournament scorer,
and bootstrap CI pipeline; (C3)~a pluggable NSGA-II integration with replaceable
optimizer interface; and (C4)~a Tournament primitive with four voting rules and
noise-stability analysis. Against a Mesa~3.3.1 baseline, HEAS requires 5
coupling lines versus 160---a 97\% reduction against Mesa's bespoke workflow and
an 88\% reduction against a competently written utility-library baseline
(42~measured lines). Three case studies---ecological, enterprise, and a
Wolf-Sheep mean-field reimplementation of a published Mesa model---validate the
framework on ODE/tabular dynamics across two simulation scales.
In the tested ecological scenarios, the evolved champion wins all 32
held-out cases and all 24 single-anchor OOD cases.
\end{abstract}

\section{INTRODUCTION}

Agent-based models are foundational for policy design across ecology,
economics, and organizational science, yet coupling an ODE-style or tabular ABM
to evolutionary optimization requires integration infrastructure that
general-purpose frameworks do not provide. This paper addresses a specific
integration failure: when researchers link an ABM to an evolutionary algorithm,
the same outcome metric is recomputed in incompatible ways across three separate
code paths---optimization, tournament evaluation, and statistical
validation---introducing silent divergence that is difficult to audit. Current
practice resolves this through bespoke coupling code (roughly 160 lines in a
representative Mesa workflow) that entangles metric definition with solver,
comparison, and inference logic.

We introduce HEAS, a framework that standardizes metric definition through a
uniform \texttt{metrics\_episode()} contract consumed identically by the
evolutionary optimizer, tournament scorer, and bootstrap confidence-interval
pipeline. This approach reduces coupling code to five lines and eliminates
conditions for metric divergence.

HEAS targets a specific niche---coupled ODE/tabular dynamics, multi-objective
evolutionary search, and multi-run validation---rather than the full ABM space
including spatially-explicit heterogeneous-agent systems. The framework's four
core components (C1--C4, Section~2) are independently useful and may be adopted
in isolation.

\subsection{The Coupling Code Problem}

Consider a researcher using Mesa to study regulatory policy in a multi-firm
economy. She wants to (1)~evolve a Pareto-optimal tax regime using NSGA-II,
(2)~compare the evolved regime against a reference policy across independent
economic scenarios using tournament voting, and (3)~report bootstrap confidence
intervals. Each step reads the same welfare metric, yet in Mesa each is an
independent code path: the DEAP fitness function extracts welfare from the
model's \texttt{DataCollector} after each run; the tournament scorer extracts
welfare from a separate episode-evaluation function, possibly with different
aggregation; and the bootstrap CI loop extracts welfare from a third data
collection loop, potentially inconsistent with both.

In practice, silent divergence is difficult to detect. Consider an experiment
where the tournament scorer reads the final step's biomass while the
evolutionary algorithm reads the episode mean biomass. The evolved champion may
become the policy with volatile-but-high peaks rather than sustainable mean
production---a consistently applied but undetected inconsistency that could
persist through publication. Such divergences are common precisely because
\emph{no single code path} connects metric definition to all three consumers.

When the researcher adds a second objective---welfare inequality---she must edit
the DataCollector reporter, the DEAP fitness function, the tournament scorer, and
the CI loop, keeping all four in sync manually. A silent divergence can persist
undetected for weeks and invalidate a publication. This pattern, which we call
\emph{coupling code}, appears in representative ABM-based policy-search
workflows~\cite{pangallo2019best,capri2023transport}.

\subsection{Contributions}

This paper's primary contribution is a validated design pattern for decoupling
metric definition from consumer logic in ODE/tabular ABM policy search. The
pattern has four separable components, each independently useful:

\textbf{C1} (Uniform contract): a \texttt{metrics\_episode()} scalar-dict
interface consumed identically by the evolutionary optimizer, tournament scorer,
and statistical inference pipeline.

\textbf{C2} (Composition model): a \texttt{Layer}/\texttt{Stream}/\texttt{Arena}
model that produces C1 without model rewrites.

\textbf{C3} (Pluggable optimizer): a reference NSGA-II wrapper with replaceable
optimizer interface supporting CMA-ES and hillclimbing.

\textbf{C4} (Tournament): a primitive implementing four standardized voting rules
with empirical noise-stability analysis.

The methodological contribution is twofold: (a)~a replicable design pattern
that reduces per-project coupling code from 160~lines to five (97\% reduction
against Mesa; 88\% against a competently written utility-library baseline) and
eliminates metric divergence risk, and (b)~a validated reference implementation
serving as both a learning artifact and a platform for extension. We evaluate
these claims through coupling-code reduction, cross-domain portability,
tournament stability, and auditability across two ecological scales and one
published ABM reimplementation.

\section{RELATED WORK}

Agent-based modeling is widely used to study complex adaptive systems in
ecology~\cite{grimm2006odd}, computational economics~\cite{tesfatsion2006ace},
and organizational behavior~\cite{bonabeau2002abm}. Macal and
North~\cite{macal2010tutorial} identify a recurring architectural division:
frameworks such as Mesa~\cite{kazil2020mesa} and NetLogo~\cite{wilensky1999netlogo}
support model execution but provide no native policy-search infrastructure. The
ODD protocol~\cite{grimm2006odd} advances ABM reproducibility through model
description, but does not address the statistical machinery needed for multi-run
policy comparison.

Simulation optimization has a rich tradition at WSC~\cite{fu2002optimization,hong2009brief}.
Fu~\cite{fu2002optimization} identifies the persistent theory-practice gap between
optimization formulations and the coupling code required to connect solvers to
simulation models. Hong and Nelson~\cite{hong2009brief} provide a practitioner-oriented
introduction that highlights the per-project engineering burden. The SimOpt
testbed~\cite{pasupathy2011simopt} catalogs benchmark problems, while commercial
platforms such as OptQuest~\cite{optquest2007} and Pagmo~\cite{biscani2020pagmo}
supply search backends---yet none address the metric-contract problem specific to
multi-objective ABM workflows. Multi-objective simulation optimization has
received growing WSC attention~\cite{hunter2022multiobjective}, and evolutionary
approaches including NSGA-II~\cite{deb2002nsga2} are increasingly deployed in
agent-based settings~\cite{capri2023transport}.

Prior ABM policy-search studies~\cite{tesfatsion2006ace,pangallo2019best,zheng2022aieconomist}
bridge the framework-optimizer gap with bespoke coupling code.
Ranking and selection methods~\cite{goldsman1994ranking} address experiment design
but assume single-objective outcomes.
Optimization libraries such as DEAP~\cite{fortin2012deap},
Optuna~\cite{akiba2019optuna}, and Pagmo~\cite{biscani2020pagmo} provide
flexible search backends but define no metric interface shared across
optimization, tournament evaluation, and statistical inference. HEAS's
\texttt{metrics\_episode()} contract closes this gap by being consumed
identically at all three stages.

Ensemble-based methods for multi-objective optimization have matured.
EMAworkbench~\cite{kwakkel2017workbench} provides simulation ensemble
management and Pareto-rank evaluation across model instances, but leaves the
metric contract unspecified---users define metrics per model, and coupling
between metric definition and ensemble aggregation remains project-specific.
Ax and BoTorch~\cite{bakshy2018ae} offer flexible multi-objective Bayesian
optimization with diversity-preserving acquisition functions, yet both target
high-dimensional continuous search and assume metrics are supplied externally,
without standardizing the pathway back to tournament evaluation or statistical
validation. Unlike these methods, HEAS does not compete with optimizer
selection; instead, it standardizes the metric interface consumed by any
optimizer---evolutionary, ensemble-based, or Bayesian. By fixing the metric
contract, HEAS reduces friction for swapping solvers in ABM workflows and
enables tournament scoring and inference to reuse the same metric pathway
without per-solver or per-model reimplementation.

HEAS is adjacent to, but distinct from, benchmark suites, optimizer wrappers,
model adapters, and reproducibility tooling: it standardizes the metric pathway
shared by optimization, tournament scoring, and inference.

\section{THE HEAS FRAMEWORK}

HEAS is implemented in Python~3.11 and depends on DEAP for evolutionary
algorithms, NumPy for numerical computation, and SciPy for statistical
utilities. All components---EA algorithm, voting rule, statistical
estimator---can be replaced without changing simulation or metric contract code.

\subsection{Layered Composition}

This subsection defines the compositional boundary that lets models be reused
without rewriting evaluation code. The fundamental abstraction is a
\texttt{Layer} with four methods
(\texttt{step}, \texttt{metrics\_step}, \texttt{metrics\_episode},
\texttt{reset}). All downstream components read the same
\texttt{metrics\_episode()} keys.

Layers compose into \textbf{Streams}---named groups forming a coherent
sub-simulation---and Streams compose into an \textbf{Arena}, which orchestrates
step ordering, inter-stream signal routing, and episode-level aggregation. The
naming convention \texttt{stream\_name.metric\_name} ensures globally unambiguous
keys regardless of Arena size. Explicit composition boundaries make components
testable and replaceable without rewriting optimization or tournament code;
the uniform metric contract (C2) remains stable as model internals evolve.

\subsection{Native Multi-Objective EA Integration}

This subsection shows how the metric contract is consumed by optimization
without introducing a separate evaluation interface. The wrapper handles DEAP's creator/toolbox registration, crossover and mutation
wiring, hall-of-fame management, and hypervolume computation. A researcher adds
NSGA-II to a new simulation by writing only the objective function (typically
5--10 lines reading from \texttt{run\_many()} results). The EA algorithm is
replaceable: \texttt{run\_ea()} also accepts CMA-ES and hillclimbing.
Section~\ref{sec:scale} shows that at 1,000-step ecological scale, NSGA-II's
advantage over random search is large ($d\!=\!1.39$), showing that
algorithm-agnosticism and algorithm quality are complementary properties.

\texttt{run\_many()} executes $n$ episodes in parallel via
\texttt{ProcessPoolExecutor}, with per-episode seeds derived deterministically
from a base seed, making individual fitness calls reproducible.

\subsection{Tournament Evaluation}

This subsection defines the comparison layer that uses the same metric contract
for multi-scenario policy ranking. Comparing candidate policies is formalized as a \texttt{Tournament}: a
cross-product of participants and scenarios evaluated for $n_\text{ep}$
episodes and aggregated by argmax, majority, Borda, or Copeland rules. The
rules are intended to reveal whether rankings are robust to aggregation choice.
Universal agreement (as in Section~5.4) indicates clear dominance;
disagreement diagnoses the competitive structure.
HEAS reports rule agreement, sample complexity $P(\text{correct})$ versus
$n_\text{ep}$, and Kendall's~$\tau$~\cite{kendall1938new} stability versus
score noise.

\subsection{Statistical Infrastructure}

This subsection makes the inference layer explicit: the same episode metrics
used by optimization and tournaments also feed uncertainty quantification.
\texttt{heas/utils/stats.py} provides: \texttt{summarize\_runs()} (mean, std,
median, BCa bootstrap 95\% CI~\cite{efron1994bootstrap}),
\texttt{wilcoxon\_test()}, \texttt{cohens\_d()}, and \texttt{kendall\_tau()}
with bootstrap CI. \texttt{heas/utils/pareto.py} provides
\texttt{hypervolume()}~\cite{zitzler1998hypervolume} and
\texttt{auto\_reference\_point()}. All utilities accept \texttt{list[float]}
and return structured dicts, requiring no configuration.

\subsection{Design Constraints and Trade-offs}
\label{sec:design-constraints}

The uniform metric contract \texttt{metrics\_episode()} deliberately implements
a minimalist interface: \texttt{dict[str, float]}. This design choice reflects a
composability-versus-expressiveness trade-off.

\textbf{Minimalism enables composition.} A richer contract---e.g.,
\texttt{dict[str, list[float]]} permitting per-step time series, or
\texttt{dict[str, dict[str, float]]} enabling agent-level distributions---would
require negotiation between model internals and the optimization, tournament, and
inference layers. By constraining the contract to scalar floats, HEAS avoids
entangling aggregation logic with metric definition. A model that produces
agent-level data aggregates once at the \texttt{metrics\_episode()} boundary, then
reuses that scalar summary everywhere downstream. Richer contracts would require
per-stage aggregation choices, reintroducing the coupling-code problem HEAS targets.

\textbf{What is lost.} The scalar contract does not directly represent time-series
outputs or agent-level heterogeneity (e.g., the distribution of firm sizes, not
just mean firm profit). For policy-ranking purposes---the intended use case---this
loss is bounded: selecting among regulatory regimes requires aggregate summaries
as the basis for comparison, even if within-regime agent-level structure remains
unmeasured.

\textbf{NSGA-II as reference implementation.} Section~3.2 adopts NSGA-II as the
default multi-objective evolutionary algorithm based on three criteria:
(1)~established performance on policy-search benchmarks in the simulation
optimization literature; (2)~well-understood parametrization at WSC; and
(3)~ease of replacement via the pluggable \texttt{run\_ea()} interface. The
framework does not depend on this choice: Section~5.4 demonstrates that algorithm
selection should be landscape-aware, and HEAS's agnostic interface enables this
flexibility without code changes.

\section{CASE STUDIES}

Three case studies span ecological, economic, and published-model domains.

\subsection{Ecological Population Management}

A predator-prey Arena with a 2-gene policy (\texttt{risk},
\texttt{dispersal}) and objectives \texttt{-agg.mean\_biomass},
\texttt{agg.cv}. Five streams model climate forcing, landscape quality, prey
logistic growth, predator dynamics, and cross-stream aggregation; eight
fragmentation$\times$shock scenarios form the evaluation grid.
Section~5.3 validates this model at two simulation scales; the evolved champion
is subjected to a 32-scenario out-of-distribution robustness test in
Section~5.4.

\subsection{Enterprise Regulatory Design}

A four-layer regulatory Arena with a 4-gene policy
(\texttt{tax\_rate}, \texttt{audit\_intensity}, \texttt{subsidy},
\texttt{penalty\_rate}) and objectives \texttt{-welfare.total},
\texttt{welfare.gini}. Firms maximize per-step profit against a stochastic
demand schedule; the regulator, government, and welfare layers implement
enforcement, redistribution, and aggregation. The Pareto trade-off is
aggregate welfare versus distributional equality across 32 governance scenarios.
Section~5.6 reports cross-domain validation at large scale.

\subsection{Wolf-Sheep Predation: Mean-Field Reimplementation}
\label{sec:wolfsheep}

To demonstrate compatibility with \emph{existing published models}, we
re-implement the canonical Mesa Wolf-Sheep Predation model~\cite{wilensky1997wolfsheep,kazil2020mesa}---the flagship example distributed with the Mesa library---as a mean-field ODE Arena.
Its published dynamics are Lotka-Volterra~\cite{lotka1925elements,volterra1926variazioni} predator-prey with grass regrowth on a
20$\times$20 spatial grid.

\textbf{HEAS reimplementation.} We implement the mean-field ODE approximation as a
4-layer Arena, deriving each term directly from Mesa's energy mechanics.
All six published parameters are used verbatim.

\textit{Grass.} Mesa's binary GrassPatch agents collapse to a continuous
density $G\in[0,1]$:
\begin{equation}
  \Delta G = \frac{1-G}{\tau_g} - \frac{S\,G\,\gamma}{N},
  \label{eq:grass}
\end{equation}
where $\tau_g\!=\!30$, $N\!=\!400$, and $\gamma\!\in\![0,1]$ is the policy
grazing-rate gene.

\textit{Sheep} ($g_s\!=\!4$, $r_s\!=\!0.04$):
\begin{equation}
  \Delta S = \Bigl(r_s - \frac{\max(0,\,1-g_s G\gamma)}{g_s}\Bigr)S
             \;-\; \frac{W\,S}{N}.
  \label{eq:sheep}
\end{equation}

\textit{Wolves} ($g_w\!=\!20$, $r_w\!=\!0.05$; $h$ = harvest-rate gene):
\begin{equation}
  \Delta W = r_w \frac{W\,S}{N}
             \;-\; \frac{\max(0,\,1-g_w S/N)}{g_w}\,W
             \;-\; h\,W.
  \label{eq:wolf}
\end{equation}

Every coefficient maps directly to a published Mesa default; no parameters
are ad-hoc. Concretely: \texttt{grass\_regrowth\_time}\,$\to\!\tau_g$,
\texttt{sheep\_gain\_from\_food}\,$\to\!g_s$,
\texttt{sheep\_reproduce}\,$\to\!r_s$,
\texttt{wolf\_gain\_from\_food}\,$\to\!g_w$,
\texttt{wolf\_reproduce}\,$\to\!r_w$,
\texttt{initial\_population}\,$\to\!N$ (density normalizer). The spatial grid
is collapsed to a density field, an acknowledged simplification (Section~6).
Adding EA optimization to this published model required zero additional coupling
code: the 8-line objective function reads \texttt{metrics\_episode()} keys,
identical in structure to the other two case studies.

A 30-run algorithm comparison on this model is reported in Section~5.5,
where the landscape structure determines which optimizer wins.

\section{EVALUATION}

\subsection{Evaluation Questions and Protocol}

We pre-specify five evaluation questions partitioned into two categories:
framework verification and domain validation. Framework verification
RQs~(RQ1,~RQ5) directly test whether HEAS achieves its core claims---that
coupling code is reduced and that the metric contract enables code-level reuse
across heterogeneous domains without modification. Domain validation
RQs~(RQ2,~RQ3,~RQ4) apply HEAS to ecological dynamics at two scales and a
published Wolf-Sheep predator-prey model, demonstrating framework usability on
realistic model families and whether evolved solutions generalize to held-out
and out-of-distribution test cases. We emphasize that domain validation RQs
establish framework applicability, not algorithmic superiority: the absence of
impediment to reuse and the absence of silent divergence are the primary
claims, not that HEAS produces better outcomes than alternative optimization
strategies. For RQ1 and RQ5 (framework verification), falsification would
require evidence that: (1)~the metric contract does not eliminate divergence
across optimization, tournament, and CI pathways, or (2)~substantial boilerplate
remains after removing model-specific logic. Each reported result is traced to
an artifact family in the appendix source list.

\subsection{Mesa Comparison: Coupling Code Reduction}
\label{sec:mesa-comparison}

We implement the ecological case study as a competent Mesa~3.3.1 model and
count non-blank, non-comment coupling code (model dynamics excluded).
Measured totals: 160~LOC (Mesa) vs.\ 5~LOC (HEAS)---97\% reduction.
To establish a fairer baseline, we additionally implement a reusable
utility-library abstraction (\texttt{mesa\_eco\_util.py}) that wraps
DataCollector setup, metric extraction, episode runners, and seed management
into a single \texttt{MesaEpisodeRunner} class. The call-site coupling code
for this utility-library approach is 42~LOC (measured)---still an 88\%
reduction compared to HEAS's 5~LOC. The utility class itself requires
$\sim$55~additional lines of one-time per-project infrastructure.

    label=lst:mesa-blocks, language=Python]
# Block A - DataCollector setup (15 LOC)
# Adding a metric: edit here AND the EA function
# AND the tournament scorer, all in sync.
self.datacollector = mesa.DataCollector(
    model_reporters={
        "prey":    lambda m: m.prey,
        "pred":    lambda m: m.pred,
        "extinct": lambda m: float(m.pred <= 0.0),
        "biomass": lambda m: m.prey + m.pred,
    }
)

# Block B - episode metric extraction (20 LOC)
# Every consumer (EA, tournament, CI) calls this
# separately. Silent divergence risk.
def extract_episode_metrics(model):
    df = model.datacollector.get_model_vars_dataframe()
    mean_biomass = float(df["biomass"].mean())
    cv = df["biomass"].std() / mean_biomass \
         if mean_biomass > 0 else 0.0
    return {"mean_biomass": mean_biomass, "cv": cv,
            "extinct": float(df["extinct"].iloc[-1])}
\end{lstlisting}

projects inherit the uniform contract without writing or maintaining a
utility module.

\subsection{Multi-Scale Reproducibility}
\label{sec:multiscale}

For RQ2, we test whether the ecological study is reproducible across the
small- and large-scale settings used in this paper. At small scale
($n\!=\!30$, pop\,=\,20, ngen\,=\,10, steps\,=\,150),
ecological HV\,=\,7.665\,(std\,3.518, CI\,[6.424,\,8.914]; HV reference point:
mean\_biomass\,=\,$-$150, CV\,=\,0.5).
The distribution is bimodal, and a $4\!\times\!4$ budget ablation
($n\!=\!20$ runs/cell; S2c) identifies population size as the controlling
factor: escape rate (HV\,$>\!9.0$) rises from 15--25\% at pop\,$\le\!20$ to
40--80\% at pop\,$\ge\!30$.

The study was then replicated at larger scale
(steps\,=\,1,000, n\_eval\,=\,5 episodes per genome evaluation, pop\,=\,30,
ngen\,=\,15, $n\!=\!20$ runs; the reduction from $n\!=\!30$ to $n\!=\!20$
reflects computational cost increase at the 1,000-step horizon, with variance
behavior confirmed robust by the 4\,$\times$\,4 budget ablation study S2c). At this horizon NSGA-II outperforms random
search: mean HV\,=\,16.66\,(std\,10.84,\,CI\,[12.37,\,20.94])
versus random mean\,=\,6.00\,(std\,0.34,\,CI\,[5.85,\,6.14]).
Wilcoxon test: $p\!=\!1.91\!\times\!10^{-6}$; Cohen's $d\!=\!1.39$
(large effect); NSGA-II achieves 178\% higher mean HV than random search.
At 150-step scale the same comparison is not significant ($d\!=\!0.49$,
$p\!=\!0.49$); the Pareto landscape becomes meaningfully richer only at the
longer horizon.

\subsection{Tournament Validation and OOD Champion Robustness}
\label{sec:tournament}

For RQ3, we evaluate stable tournament behavior and champion performance in the
tested ecological scenario families. Voting-rule agreement and sample complexity
were assessed across 30 repeats of
the 8-scenario tournament with 100 episodes per scenario. All four voting rules
agreed in 100\% of
(scenario, repeat) pairs, and $P(\text{correct winner})\!=\!1.0$ at all tested
episode budgets (4, 10, 25, 50, 100 per scenario). The 100\% agreement likely
reflects clear dominance on these low-dimensional problems: with only 2--4 policy
genes and 8 scenarios, the Pareto landscape offers unambiguous winners. A
near-tie sensitivity study~(S9) confirms that this unanimity disappears when
candidate policies have statistically indistinguishable means.

Noise stability was assessed by injecting synthetic Gaussian noise
$\mathcal{N}(0,\sigma^2)$ into episode scores (30 repeats) and measuring
Kendall's $\tau$~\cite{kendall1938new} between clean and noisy rankings. This
synthetic perturbation serves as a proxy for real stochasticity in outcome
metrics; it characterizes the tournament's mathematical sensitivity to score
noise rather than measuring variance from specific simulation runs. Rankings are stable ($\tau\!=\!1.0$) for noise
up to 0.65\% of the inter-policy margin: $\tau\!=\!0.944$ at $\sigma\!=\!10$
(6.5\% of margin), degrading to $\tau\!=\!0.744$ at $\sigma\!=\!50$.

\begin{figure}[ht]
  \centering
  \includegraphics[width=0.846\columnwidth]{fig8_noise_stability}
  \caption{Kendall's $\tau$ versus injected score noise $\sigma$ ($n\!=\!30$
    repeats, 8 scenarios). Rankings are stable ($\tau\!=\!1.0$) until noise
    exceeds $\sim$6\% of the inter-policy margin.}
  \label{fig:noise}
\end{figure}

OOD champion robustness was validated at two scales. At small scale (16
scenarios, steps\,=\,500), the champion won all 16 ($p\!<\!0.001$). At large
scale (32 scenarios), the champion policy
won all 32 ($\Delta\!=\!+25.0$, $+3.96\%$; $p\!=\!4.66\!\times\!10^{-10}$;
$d\!=\!0.17$). The small effect size ($d\!=\!0.17$) reflects near-deterministic
score distributions in these ecological scenarios; win rate and paired deltas
are the more informative statistics here, as score variance is low across
scenarios. In the single-anchor \emph{true OOD} study (Exp.~C, S6c), the
champion won all 24 scenarios where all four factors lay outside training
($p\!=\!1.19\!\times\!10^{-7}$), matching the 100\% in-distribution win rate.
Here win rate and paired deltas are more informative than Cohen's $d$ because
the score distributions are nearly deterministic.

\subsection{Algorithm Agnosticism and Scale Sensitivity}
\label{sec:scale}

For RQ4, we test whether HEAS remains useful when the best optimizer depends on
landscape structure. An algorithm ablation at steps\,=\,300
(pop\,=\,20, ngen\,=\,10, $n\!=\!10$ runs)
finds that Simple hillclimbing outperforms NSGA-II (means: Simple\,=\,19.66,
NSGA-II\,=\,9.99, Random\,=\,7.61). On a 2-gene landscape the Pareto front is
effectively one-dimensional; single-objective search concentrates population
on the biomass-maximizing direction while NSGA-II distributes across the CV
trade-off. This reinforces the agnosticism claim: HEAS allows algorithm
selection without any framework changes.

Scale sensitivity is reported in Figure~\ref{fig:heatmap}, which maps mean HV
across a 3\,$\times$\,3 grid of steps\,$\in$\,\{140, 300, 500\} and episodes per
evaluation\,$\in$\,\{5, 10, 20\} ($n\!=\!10$ runs each; HV reference point:
mean\_biomass\,=\,$-$200, CV\,=\,0.5, consistent with the large-scale study). HV rises from
6.0 (140 steps, 5 episodes) to 10.1 (300 steps, 20 episodes) within the
small-scale regime, confirming that longer episodes yield richer Pareto
landscapes. The bimodal variance pattern persists across all cells (consistent with the
population-budget ablation findings in Section~\ref{sec:multiscale}).

\begin{figure}[ht]
  \centering
  \includegraphics[width=0.846\columnwidth]{fig3_scale_heatmap}
  \caption{Scale sensitivity heatmap: mean HV by simulation horizon
    (steps\,$\times$\,episodes/genome). Each cell is $n\!=\!10$ independent
    NSGA-II runs. HV generally increases with steps; non-monotonicities
    between the 300- and 500-step rows reflect high between-run variance.
    Episode count has a weaker secondary effect.}
  \label{fig:heatmap}
\end{figure}

The Wolf-Sheep algorithm comparison (steps\,=\,200, pop\,=\,50, ngen\,=\,25,
$n\!=\!30$ runs) yields Random\,=\,14.35 (std\,0.61), NSGA-II\,=\,10.36
(std\,4.07), and Simple\,=\,9.14 (std\,3.68). Random achieves the highest
HV; a two-sided Wilcoxon rank-sum test confirms the difference is significant
($p\!=\!7.99\!\times\!10^{-6}$, Random\,$>$\,NSGA-II). On this landscape,
simple hillclimbing substantially outperforms NSGA-II because the landscape
is effectively one-dimensional: with only two genes (harvest rate and grazing
rate), the Pareto front forms a narrow band of sustainable combinations,
making diversity preservation a liability rather than an asset. NSGA-II's
computational overhead thus becomes a handicap compared to the greedy
efficiency of local ascent---a pattern that disappears on richer multi-objective
landscapes (see Section~\ref{sec:multiscale}).

Algorithm-landscape matching---made practical by C3's pluggable optimizer
interface---is the operational takeaway: on simple or single-equilibrium
landscapes any search strategy suffices; on complex multi-objective
landscapes NSGA-II's structural advantage is large ($d\!=\!1.39$).

\subsection{Cross-Domain Portability}
\label{sec:crossdomain}

For RQ5, we ask whether the same framework code can be
reused across domains at scale. We report the large-scale ecological and
enterprise portability runs from S8, and keep the separate small-scale
enterprise validation (S2b) out of this comparison. The same codebase---with no
changes to framework code beyond domain-specific \texttt{metrics\_episode()}
implementations---was run on two heterogeneous domains. In the ecological domain
(steps\,=\,1,000), NSGA-II achieves mean HV\,=\,15.46
(std\,8.57, CI\,[12.03,\,18.88]); the bimodal convergence pattern persists. In
the enterprise domain (steps\,=\,200, $n\!=\!20$), NSGA-II achieves mean
HV\,=\,$5.565\!\times\!10^{6}$ (std\,$6.96\!\times\!10^{4}$,
CI\,$[5.535,\,5.594]\!\times\!10^{6}$). The portability claim is therefore
code-level rather than outcome-level: the same framework interfaces support both
domains without rewriting optimization, tournament, or statistics code.
Tournament validation was also conducted in the enterprise
domain across 8 governance scenarios (20 repeats, 30 episodes/scenario)~(S10).
All four voting rules agreed in 100\% of (scenario, repeat) pairs, and
$P(\text{correct winner})\!=\!1.0$ at all tested episode budgets (4, 10, 25,
50), showing that the tournament infrastructure transfers across these tested
domains without framework modification. The portability validation is scoped to
coupled ODE/tabular dynamics; independent evaluation in spatially-explicit
agent-based models, network-based systems, or heterogeneous-agent interaction
patterns remains future work and lies outside HEAS's current design scope.

\subsection{Threats to Validity}

\textbf{Construct validity.} The case studies are composition vehicles, not
calibrated domain models; results are evidence about framework behavior. The
metric contract prevents \emph{accidental divergence across consumers}: all
three pipeline stages read the same key. An incorrect \texttt{metrics\_episode()}
implementation will be consistently wrong rather than silently inconsistent. The
framework additionally exposes \texttt{validate\_metrics\_episode()}, which raises
\texttt{TypeError} when a \texttt{metrics\_episode()} implementation returns a
non-dict or non-float value, catching contract violations early in development.

\textbf{Internal validity.} Fixed evaluation seeds isolate framework effects
but can suppress variance in some metrics; zero-variance outcomes are treated as
diagnostic signals (e.g., the $K\!=\!120$ degeneracy, enterprise determinism)
rather than performance claims. OOD validation at steps\,=\,500 and on
single-anchor extrapolation scenarios (Exp.~C) extends evaluation beyond the
training horizon, but only within the tested family of ecological scenarios.

\textbf{External validity.} The Mesa baseline is a competent single-project
implementation, not a census of all coding styles. Runtime findings for
\texttt{n\_jobs} depend on episode cost and hardware. All three case studies
use ODE-style or tabular dynamics; researchers working with genuinely
spatially-explicit ABMs---heterogeneous agents on a grid, emergent network
effects, rich move-and-interact loops---will encounter integration patterns
outside HEAS's current design scope. The 97\% LOC reduction is validated
specifically for the tabular policy-search workflow that HEAS targets. The
results therefore support HEAS on the tested model family, not on arbitrary ABM
domains.

\section{CONCLUSION}

These experiments support a narrower claim: HEAS makes simulation-based policy
search more auditable and reusable by enforcing one metric contract across
optimization, comparison, and inference. Across five evaluation questions, the
paper shows 97\% coupling-code reduction against the specific Mesa baseline
(160 vs.\ 5 LOC), reproducible ecological behavior at two scales, stable
tournament rankings under realistic noise ($\tau\!>\!0.94$), and algorithm
flexibility when landscape structure favors simpler optimizers. Three case
studies---including a mean-field compatibility demonstration based on Mesa's
published Wolf-Sheep model---show how the same framework interfaces can be
reused across the tested model family. Within the tested ecological scenario
families, the evolved champion wins all 32 held-out cases and all 24
single-anchor OOD cases. A $4\!\times\!4$ budget ablation explains the observed
bimodal convergence as a diversity-threshold transition around pop\,=\,30.

\textbf{Limitations.} For lightweight ODE simulations ($<$0.01\,s/episode),
process startup overhead ($\approx$9\,s) dominates; use \texttt{n\_jobs=1}.
HEAS is designed for ODE-style and tabular simulations; spatially-explicit ABMs
remain Mesa's domain. The metric contract is scalar-focused
(\texttt{dict[str, float]}); workflows requiring agent-level distributions need
an additional aggregation layer. In the enterprise inequality case study, HEAS
computes Gini via step-wise scalar aggregates from the ABM layer---each
simulation step reports a single Gini coefficient across all firms. This loses
agent-level heterogeneity (the firm-level inequality distribution, not just mean
Gini across steps). For policy-search purposes this loss is acceptable:
ranking regulatory regimes by distributional equity requires only the regime's
average Gini as a summary statistic, even if within-regime firm-level inequality
structure remains unmeasured. Extensions to the contract would be needed for
workflows characterizing within-agent distributions, such as sensitivity analysis
on the firm-size distribution conditional on regime.

\textbf{Future work.} Priorities are a \texttt{MesaLayerAdapter} (wrapping existing
Mesa models as HEAS Layers without rewriting dynamics), scalable parallel backends
(Ray/Dask), and automated scenario generation.
